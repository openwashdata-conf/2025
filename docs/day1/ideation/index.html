Team Formation Process

1. Problem Identification (20 min)

  • Review challenges in WASH data
  • Identify personal interests and skills
  • Consider available resources

2. Solution Brainstorming (30 min)

  • Generate multiple solutions per challenge
  • Use rapid prototyping sketches
  • Consider technology requirements

3. Project Pitches (30 min)

  • 2-minute pitches per idea
  • Clear problem statement
  • Proposed solution approach
  • Required team skills

4. Team Formation (20 min)

  • Match skills to project needs
  • Balance team sizes (3-5 people)
  • Exchange contact information
  • Designate project lead

Good Hackathon Projects

Projects should be:

  • Achievable: Can be completed in one day
  • Relevant: Addresses real openwashdata needs
  • Collaborative: Requires diverse skills
  • Demonstrable: Has clear outputs to showcase
  • Sustainable: Can be continued after the hackathon
